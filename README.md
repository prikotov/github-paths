# GitHub Paths

> Популярные страницы репозиториев из GitHub API

## Зачем это нужно

Этот skill показывает, какие страницы и файлы вашего репозитория просматривают чаще всего. Данные помогают:

- **Понять интересы аудитории** — какие файлы/документы востребованы
- **Оптимизировать структуру** — переработать популярные страницы
- **Улучшить документацию** — дополнить часто просматриваемые файлы
- **Оценить эффект изменений** — как изменения влияют на просмотры

## Что вы получите

Отчёт в форматах CSV и Markdown:

| Файл | Содержание |
|------|------------|
| `github_pages.*` | Популярные страницы: путь, заголовок, просмотры, уникальные |

## Зависимости

Skill совместим с различными AI-агентами. Примеры ниже даны для OpenCode — для других инструментов смотрите их документацию по установке skills.

## Использование

### Напрямую через PHP

```bash
# Базовые запросы
php .opencode/skills/github-paths/paths.php                    # топ-20 для default_repo
php .opencode/skills/github-paths/paths.php --repo my-project  # конкретный репозиторий

# С параметрами
php .opencode/skills/github-paths/paths.php -l 50              # топ-50 страниц
php .opencode/skills/github-paths/paths.php --sort unique      # сортировка по уникальным
```

### Параметры

| Параметр | Сокращение | Описание | Пример |
|----------|------------|----------|--------|
| `--repo` | `-r` | Репозиторий из конфига | `--repo my-project` |
| `--limit` | `-l` | Лимит записей | `-l 50` |
| `--sort` | `-s` | Поле сортировки: views, unique | `-s unique` |

### Ограничения API

⚠️ **GitHub API ограничения:**
- Данные доступны только за последние 14 дней
- Popular paths — максимум 10 записей
- Rate limit: 5000 запросов/час

### Через агента

После установки skill агент автоматически узнаёт о нём. Примеры запросов:

```
Какие страницы репозитория самые популярные?
```

```
Покажи топ файлов по просмотрам для playbook
```

```
Какие файлы в репозитории чаще всего просматривают уникальные посетители?
```

```

```

CSV открывается в Excel/LibreOffice, Markdown — в любом текстовом редакторе или напрямую в Obsidian.
